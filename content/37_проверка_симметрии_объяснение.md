# Задача: Проверка центральной симметрии

> **Уровень сложности:** Medium  
> **Тема:** HashSet, геометрия  
> **Встречается на:** Яндекс, Google

---

## 1. Что просят?

```
Проверить, является ли набор точек центрально-симметричным.
Набор симметричен относительно точки C если для каждой точки P
существует точка P' такая что C — середина отрезка PP'.

Центр C вычисляется как среднее всех точек.

Примеры:
  [(1,0),(0,1),(-1,0),(0,-1)] → true  (центр (0,0))
  [(0,0),(2,0)]               → true  (центр (1,0))
  [(0,0),(1,0),(2,1)]         → false
```

---

## 2. Ключевая идея

1. Центр симметрии C = (sumX/n, sumY/n)
2. Для каждой точки (x,y) симметричная точка: (2*cx-x, 2*cy-y)
3. Проверить что симметричная точка есть в наборе (HashSet → O(1))

Чтобы избежать дробей: работать с `2*sum` и проверять делимость на n.

---

## 3. Пошаговый алгоритм

```
n = кол-во точек
если n нечётное: false (нечётное кол-во точек не может быть симметрично)
sumX = sum(x_i), sumY = sum(y_i)
если 2*sumX % n != 0 или 2*sumY % n != 0: false (центр нецелый)

doubleCX = 2*sumX/n, doubleCY = 2*sumY/n
pointSet = {все точки}

Для каждой точки (x,y):
  symm = (doubleCX - x, doubleCY - y)
  если symm не в pointSet: false

Вернуть true
```

---

## 4. Трассировка

**[(1,0),(0,1),(-1,0),(0,-1)]**

```
n=4, sumX=0, sumY=0
doubleCX = 0, doubleCY = 0

Проверяем:
  (1,0) → симм=(0-1, 0-0)=(-1,0) → есть ✓
  (0,1) → симм=(0, -1) → есть ✓
  (-1,0)→ симм=(1, 0) → есть ✓
  (0,-1)→ симм=(0, 1) → есть ✓

Ответ: true ✓
```

| | Сложность |
|---|---|
| Время | O(n) |
| Память | O(n) — HashSet |

---

## 6. Типичные ошибки

**Ошибка 1: Не проверить нечётное количество точек.**
Нечётное число точек не может быть попарно симметричным — сразу `false`.

**Ошибка 2: Использовать дробные координаты центра.**
Работать с `2*sumX` / n и проверять делимость — избегаем float.

**Ошибка 3: Неправильно вычислять симметричную точку.**
Если центр (cx,cy), то симметричная к (x,y): `(2*cx - x, 2*cy - y)`.
