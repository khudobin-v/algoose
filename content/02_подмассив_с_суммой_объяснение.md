# Задача: Подотрезок с суммой X

> **Уровень сложности:** Medium  
> **Тема:** Prefix Sum, HashMap, скользящее окно  
> **Аналог на LeetCode:** 560. Subarray Sum Equals K  
> **Встречается на:** Яндекс, Google, Facebook

---

## 1. Что просят?

```
Дан массив целых чисел (могут быть отрицательные!) и число X.
Найти непрерывный подмассив с суммой = X.
Вернуть индексы начала и конца (или сам подмассив).

Примеры:
  [1, 2, 3, 7, 5], X=12  →  [2, 3, 7] (индексы 1-3)
  [1, -1, 5, -2, 3], X=3  →  [1, -1, 5, -2] (индексы 0-3)
  [1, 2, 3], X=10  →  нет решения
```

> ⚠️ Сразу уточни у интервьюера: числа только положительные или могут быть отрицательные?  
> Это меняет алгоритм!

---

## 2. Два разных алгоритма

### Случай 1: Только положительные числа → Скользящее окно O(n)

Если все числа положительные, то при расширении окна сумма только растёт,
при сужении — только уменьшается. Можно двигать два указателя.

### Случай 2: Есть отрицательные → Prefix Sum + HashMap O(n)

Скользящее окно не работает — расширение может уменьшить сумму.
Нужен другой подход.

**Ключевая идея Prefix Sum:**  
Если `prefix[j] - prefix[i] = X`, то подмассив `[i+1, j]` имеет сумму X.  
Значит нужно найти два индекса где разница prefix-сумм равна X.

---

## 3. Код на Dart — только положительные (скользящее окно)

```dart
(int, int)? subarraySumPositive(List<int> a, int target) {
  int left = 0;
  int currentSum = 0;

  for (int right = 0; right < a.length; right++) {
    currentSum += a[right];

    // Сумма слишком большая — сжимаем окно слева
    while (currentSum > target && left < right) {
      currentSum -= a[left];
      left++;
    }

    if (currentSum == target) return (left, right);
  }

  return null; // не найдено
}
```

---

## 4. Код на Dart — с отрицательными (prefix sum + HashMap)

```dart
(int, int)? subarraySum(List<int> a, int target) {
  // Храним: prefix_sum → индекс где эта сумма была достигнута
  final seen = <int, int>{0: -1}; // сумма 0 "была" до начала массива
  int prefixSum = 0;

  for (int i = 0; i < a.length; i++) {
    prefixSum += a[i];

    // Ищем: был ли prefix[j] = prefixSum - target?
    final needed = prefixSum - target;
    if (seen.containsKey(needed)) {
      return (seen[needed]! + 1, i);
    }

    // Сохраняем только первое вхождение (для минимального индекса)
    if (!seen.containsKey(prefixSum)) {
      seen[prefixSum] = i;
    }
  }

  return null;
}
```

## 5. Трассировка — с отрицательными

**Вход:** `[1, -1, 5, -2, 3]`, `target=3`

```
seen = {0: -1}
prefixSum = 0

i=0: a[0]=1, prefixSum=1, needed=1-3=-2, нет в seen. seen={0:-1, 1:0}
i=1: a[1]=-1, prefixSum=0, needed=0-3=-3, нет в seen. seen={0:-1, 1:0}  (0 уже есть!)
i=2: a[2]=5, prefixSum=5, needed=5-3=2, нет в seen. seen={0:-1,1:0,5:2}
i=3: a[3]=-2, prefixSum=3, needed=3-3=0, ЕСТЬ в seen! seen[0]=-1
  → возврат (-1+1, 3) = (0, 3)

Подмассив [0..3] = [1,-1,5,-2], сумма = 3 ✓
```

---

## 6. Трассировка — скользящее окно

**Вход:** `[1, 2, 3, 7, 5]`, `target=12`

```
left=0, sum=0

right=0: sum=1, sum<12
right=1: sum=3, sum<12
right=2: sum=6, sum<12
right=3: sum=13, sum>12 → сжимаем: sum-=a[0]=1→sum=12, left=1
  sum==12 → ответ (1, 3) = [2,3,7] ✓
```

## 7. Частые ошибки

**Ошибка 1: Применить скользящее окно к массиву с отрицательными.**
```
[3, -1, 2], target=2:
right=2: sum=4, сжимаем → sum=1, сжимаем → sum=2... 
но индекс уже неверный!
```

**Ошибка 2: Не добавить `{0: -1}` в seen при старте.**  
Если весь массив с начала даёт нужную сумму — без этого не найдём.

**Ошибка 3: Перезаписывать уже сохранённый индекс в seen.**  
```dart
// Плохо — теряем первое вхождение
seen[prefixSum] = i;

// Хорошо — сохраняем первое
if (!seen.containsKey(prefixSum)) seen[prefixSum] = i;
```
