# Задача: Найти пересекающиеся встречи

> **Уровень сложности:** Medium  
> **Тема:** Интервалы, сортировка, sweep line  
> **Аналог на LeetCode:** 252. Meeting Rooms  
> **Встречается на:** Яндекс, Google

---

## 1. Что просят?

```
Дан список встреч (from, to). Найти все встречи, которые пересекаются
хотя бы с одной другой.

Примеры:
  [(1,3),(2,4),(5,7)] → [(1,3),(2,4)]   — первые две пересекаются в [2,3]
  [(1,2),(3,4)]       → []               — никто не пересекается
```

---

## 2. Ключевая идея

**Наивно O(n²):** для каждой пары проверяем пересечение.
**Оптимально O(n log n):** сортируем по началу + sweep line.
Встречи НЕ пересекаются если `m1.to <= m2.from` или `m2.to <= m1.from`.

---

## 3. Условие пересечения

```
m1 = (from1, to1)
m2 = (from2, to2)

НЕ пересекаются: to1 <= from2  ИЛИ  to2 <= from1
Пересекаются:    !(to1 <= from2 ИЛИ to2 <= from1)
               = to1 > from2 И to2 > from1
```

---

## 4. Алгоритм sweep line

```
1. Сортируем по from
2. Поддерживаем список "активных" встреч (ещё не закончились)
3. Для каждой новой встречи:
   а. Убираем из active все встречи с to <= from текущей
   б. Если active непустой — все в active пересекаются с текущей
   в. Добавляем текущую в active
```

---

## 5. Сложность

| Алгоритм | Время | Память |
|---|---|---|
| Наивный | O(n²) | O(n) |
| Sweep line | O(n log n) | O(n) |

---

## 6. Типичные ошибки

**Ошибка 1: Неправильное условие пересечения.**
```
// Плохо — не учитывает касание границ
m1.from < m2.to && m2.from < m1.to

// Граница — уточни у интервьюера:
// Если встречи [1,2] и [2,3] считаются пересекающимися → используй <=
// Если нет (касание = не пересечение) → используй <
```

**Ошибка 2: Добавлять дубликаты в результат.**
Использовать Set вместо List.
