# Задача: Наименьший общий предок (LCA) в дереве с parent-указателем

> **Уровень сложности:** Medium  
> **Тема:** Деревья, два указателя  
> **Аналог на LeetCode:** 236. Lowest Common Ancestor of a Binary Tree  
> **Встречается на:** Яндекс, Google, Meta

---

## 1. Читаем условие — что реально просят?

```
Дано бинарное дерево (НЕ дерево поиска — просто дерево).
У каждого узла есть: левый потомок, правый потомок, и указатель на РОДИТЕЛЯ.

Написать функцию lca(a, b) которая вернёт наименьшего общего предка двух узлов.

Ограничение по памяти: O(1)
```

**Наименьший общий предок (LCA)** — это самый нижний узел дерева, который является предком обоих узлов одновременно. Узел считается предком самого себя.

### Пример:
```
        3
       / \
      5   1
     / \ / \
    6  2 0  8
      / \
     7   4
```

- `lca(6, 4)` → **5** (оба живут в поддереве 5)
- `lca(5, 1)` → **3** (корень — единственный общий предок)
- `lca(5, 4)` → **5** (узел 5 является предком самого себя)

## 2. Как думать на собесе — пошагово

### Шаг 1: Задай уточняющий вопрос

Сразу скажи интервьюеру:

> "Уточню: у узлов есть указатель на родителя? Это сильно меняет подход."

Если есть — задача решается без рекурсии по дереву вниз, только движением вверх.  
Если нет — нужен рекурсивный DFS (другая задача).

---

### Шаг 2: Предложи наивное решение

Не молчи и не думай сразу про оптимальное. Покажи, что ты умеешь думать итеративно:

> "Первая идея: пройти от узла `a` вверх до корня, сохранить все узлы в Set. Потом пройти от `b` вверх — первый узел, который есть в Set, и есть LCA."

```dart
// НАИВНОЕ решение — O(h) памяти, не подходит по условию
Node? lcaNaive(Node? a, Node? b) {
  final visited = <Node>{};
  
  Node? cur = a;
  while (cur != null) {
    visited.add(cur);
    cur = cur.parent;
  }
  
  cur = b;
  while (cur != null) {
    if (visited.contains(cur)) return cur;
    cur = cur.parent;
  }
  
  return null;
}
```

Потом сам скажи почему это не подходит:

> "Но это O(h) памяти под Set. Условие требует O(1). Нужно что-то лучше."

Это показывает, что ты понимаешь ограничения — интервьюер это ценит.

---

### Шаг 3: Найди аналогию

Вот ключевой инсайт — задача очень похожа на другую классическую задачу:

> **Найти точку пересечения двух связных списков.**

Почему похожа? Потому что пути от `a` до корня и от `b` до корня — это два "списка", которые **сходятся в одной точке** (LCA), а потом идут вместе до корня.

```
a → ... → LCA → ... → root
b → ... → LCA → ... → root
               ↑
          здесь они совпадают
```

---

### Шаг 4: Вспомни решение задачи про пересечение списков

В задаче про пересечение двух связных списков есть элегантный трюк с двумя указателями:

- Запускаем два указателя — один с головы первого списка, другой со второго
- Когда указатель доходит до конца — перенаправляем его на голову **другого** списка
- Они встретятся ровно на точке пересечения

**Почему?** Оба указателя пройдут одинаковое расстояние:
```
указатель 1: [путь от a до LCA] + [путь от b до LCA]
указатель 2: [путь от b до LCA] + [путь от a до LCA]
```
Суммы одинаковы → они встретятся в LCA.

---

### Шаг 5: Перенеси решение на нашу задачу

Вместо "конца списка" — `null` (дошли выше корня).  
Вместо "головы другого списка" — стартовый узел второго.

```
p1 идёт: a → ... → root → null → b → ... → LCA
p2 идёт: b → ... → root → null → a → ... → LCA
```

## 3. Финальный код на Dart

### Структура узла:
```dart
class Node {
  int val;
  Node? parent;
  Node? left;
  Node? right;

  Node(this.val, {this.parent, this.left, this.right});
}
```

### Решение 1 — Трюк с двумя указателями (O(1) памяти):
```dart
Node? lca(Node? a, Node? b) {
  Node? p1 = a;
  Node? p2 = b;

  while (p1 != p2) {
    // Если дошли до null — переходим на старт второго узла
    p1 = p1 != null ? p1.parent : b;
    p2 = p2 != null ? p2.parent : a;
  }

  return p1;
}
```

### Решение 2 — Выравнивание глубин (проще объяснять вслух):
```dart
int _depth(Node? node) {
  int d = 0;
  while (node != null) {
    node = node.parent;
    d++;
  }
  return d;
}

Node? lca(Node? a, Node? b) {
  int da = _depth(a);
  int db = _depth(b);

  // Поднимаем более глубокий узел до одного уровня
  while (da > db) {
    a = a!.parent;
    da--;
  }
  while (db > da) {
    b = b!.parent;
    db--;
  }

  // Теперь оба на одном уровне — идём вместе до встречи
  while (a != b) {
    a = a!.parent;
    b = b!.parent;
  }

  return a;
}
```

## 4. Разбор на примере вручную

Дерево:
```
        3
       / \
      5   1
     / \
    6   2
```

**Пример: lca(6, 1), ответ = 3**

Трассируем решение 1 (два указателя):

| Шаг | p1   | p2   |
|-----|------|------|
| 0   | 6    | 1    |
| 1   | 5    | 3    |
| 2   | 3    | null → переходим на a=6 |
| 3   | null → переходим на b=1 | 5 |
| 4   | 1    | 3    |
| 5   | 3    | **3** → встретились! |

**Ответ: 3 ✓**

---

Трассируем решение 2 (выравнивание глубин):

```
Глубина 6 = 3 (6→5→3→null... считаем шаги до null)
Глубина 1 = 2 (1→3→null)

Поднимаем 6 на 1 уровень: 6 → 5
Теперь оба на глубине 2: p1=5, p2=1

Идём вместе:
  p1=5, p2=1  → не равны
  p1=3, p2=3  → равны! Ответ: 3 ✓
```

## 5. Частые ошибки на собесе

**Ошибка 1:** Сразу лезть в рекурсию вниз по дереву.
> Когда есть `parent` — всегда думай про движение **вверх**, не вниз.

**Ошибка 2:** Забыть про случай когда один узел является предком другого.
> Например `lca(5, 4)` должен вернуть 5. Оба алгоритма выше это корректно обрабатывают — проверь на примере сам.

**Ошибка 3:** В решении с выравниванием — считать глубину как расстояние от корня, а не количество шагов до null.
> Неважно как считаешь, главное считай одинаково для обоих узлов.

**Ошибка 4:** Паниковать когда интервьюер говорит "а теперь без parent-указателя".
> Тогда это другая задача — рекурсивный DFS. Скажи это вслух и предложи новый подход.

## 7. Сложность

| Решение              | Время  | Память |
|----------------------|--------|--------|
| Наивное (Set)        | O(h)   | O(h)   |
| Два указателя        | O(h)   | **O(1)** |
| Выравнивание глубин  | O(h)   | **O(1)** |

где `h` — высота дерева. В сбалансированном дереве h = O(log n), в худшем случае h = O(n).

## 8. Follow-up вопросы от Яндекса

Яндекс любит усложнять — будь готов:

**"А если parent-указателя нет?"**  
→ Рекурсивный DFS: если нашли p или q — возвращаем вверх. LCA там где оба потомка вернули не null.

**"А если это не бинарное дерево, а у узла N детей?"**  
→ Логика та же, меняется только структура — нужен обход всех детей.

**"А если нужно отвечать на M запросов LCA быстро?"**  
→ Алгоритм Тараяна или Sparse Table + Euler tour — это уже Hard уровень.
