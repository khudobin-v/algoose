# Задача: RLE сжатие строки

> **Уровень сложности:** Easy-Medium  
> **Тема:** Строки, один проход  
> **Аналог на LeetCode:** 443. String Compression  
> **Встречается на:** Яндекс (очень часто), все крупные компании

---

## 1. Что просят?

```
Дана строка из букв A-Z.
Написать функцию RLE которая заменяет каждую группу одинаковых символов
на символ + количество повторений.
Если символ встречается один раз — он остаётся без числа.

"AAABBC"     →  "A3B2C"
"ABCD"       →  "ABCD"      (каждый символ по одному разу)
"AAAAAAAAA"  →  "A9"
""           →  ошибка / пустая строка
```

> ⚠️ Уточни у интервьюера: одиночный символ писать как "A" или "A1"?  
> Стандарт RLE — писать "A1", но в задаче Яндекса обычно просят "A" (без единицы).

---

## 2. Как думать на собесе

### Шаг 1: Переформулируй
> "Мне нужно пройти строку слева направо, отслеживать текущий символ и считать сколько раз он повторяется подряд. Когда символ меняется — записываю предыдущую группу."

### Шаг 2: Алгоритм
```
Завести: currentChar = s[0], count = 1
Идти с i=1 до конца:
  если s[i] == currentChar → count++
  иначе → записать currentChar + (count > 1 ? count : ""), 
           обновить currentChar = s[i], count = 1
После цикла: записать последнюю группу (её часто забывают!)
```

### Шаг 3: Граничные случаи
- Пустая строка — вернуть ошибку или ""
- Строка из одного символа — вернуть "A"
- Все символы одинаковые — "AAAA" → "A4"
- Все символы разные — "ABCD" → "ABCD"

## 3. Код на Dart

```dart
String rleEncode(String s) {
  if (s.isEmpty) throw ArgumentError('Input string is empty');

  final result = StringBuffer();
  int count = 1;

  for (int i = 1; i <= s.length; i++) {
    if (i < s.length && s[i] == s[i - 1]) {
      count++;
    } else {
      result.write(s[i - 1]);
      if (count > 1) result.write(count);
      count = 1;
    }
  }

  return result.toString();
}
```

---

## 4. Трассировка

**Вход:** `"AAABBC"`

```
i=1: s[1]='A' == s[0]='A'  → count=2
i=2: s[2]='A' == s[1]='A'  → count=3
i=3: s[3]='B' ≠ s[2]='A'  → пишем "A3", count=1
i=4: s[4]='B' == s[3]='B'  → count=2
i=5: s[5]='C' ≠ s[4]='B'  → пишем "B2", count=1
i=6: конец строки          → пишем "C"

Результат: "A3B2C" ✓
```

---

## 5. Частые ошибки

**Ошибка 1: Забыть записать последнюю группу.**  
Цикл заканчивается — последняя группа не записана. Решение: условие `i <= s.length` вместо `i < s.length`.

**Ошибка 2: Писать "A1" вместо "A".**  
Уточни у интервьюера заранее.

**Ошибка 3: Конкатенация строк в цикле.**  
```dart
// Плохо — O(n²) из-за создания новых строк на каждой итерации
result += s[i];

// Хорошо — O(n)
final result = StringBuffer();
result.write(s[i]);
```

## 7. Follow-up

**"А как декодировать обратно?"**
```dart
String rleDecode(String s) {
  final result = StringBuffer();
  int i = 0;
  while (i < s.length) {
    final char = s[i];
    i++;
    int count = 0;
    while (i < s.length && s[i].contains(RegExp(r'[0-9]'))) {
      count = count * 10 + int.parse(s[i]);
      i++;
    }
    result.write(char * (count == 0 ? 1 : count));
  }
  return result.toString();
}
```

**"А если число может быть больше 9?"**  
→ Тогда при декодировании нужно читать несколько цифр подряд (уже учтено выше).
