# Задача: Оставить не более N повторений каждого числа

> **Уровень сложности:** Easy-Medium  
> **Тема:** HashMap, один проход  
> **Аналог на LeetCode:** 80. Remove Duplicates from Sorted Array II  
> **Встречается на:** Яндекс, Google

---

## 1. Что просят?

```
Оставить только первые N вхождений каждого числа, сохранив порядок.

Примеры:
  keepAtMost([1,1,1,2,2,3], 2) → [1,1,2,2,3]
  keepAtMost([1,2,3,1,2,1], 2) → [1,2,3,1,2]
  keepAtMost([1,1,1,1], 1)     → [1]
```

---

## 2. Ключевая идея

HashMap счётчиков. Для каждого элемента: если его счётчик < N — добавляем и увеличиваем счётчик. Иначе пропускаем.

---

## 3. Пошаговый алгоритм

```
count = {}

Для каждого числа x:
  если count[x] < N:
    result.add(x)
    count[x]++
```

---

## 4. Трассировка

**nums=[1,2,3,1,2,1], N=2**

```
x=1: count[1]=0 < 2 → add(1), count[1]=1
x=2: count[2]=0 < 2 → add(2), count[2]=1
x=3: count[3]=0 < 2 → add(3), count[3]=1
x=1: count[1]=1 < 2 → add(1), count[1]=2
x=2: count[2]=1 < 2 → add(2), count[2]=2
x=1: count[1]=2 >= 2 → пропускаем

Результат: [1,2,3,1,2] ✓
```

---

## 5. Особый случай: отсортированный массив

Для отсортированного массива можно без HashMap — O(1) памяти:
```dart
// Принимаем элемент если он != элементу на maxCount позиций назад
if (writePos < maxCount || nums[i] != nums[writePos - maxCount]):
  nums[writePos++] = nums[i]
```

---

## 6. Сложность

| | Время | Память |
|---|---|---|
| Произвольный (HashMap) | O(n) | O(k) — k уникальных значений |
| Отсортированный | O(n) | O(1) |

---

## 7. Типичные ошибки

**Ошибка 1: Изменять массив во время обхода — пропуск элементов.**
Создавать новый список или использовать writePos-паттерн.

**Ошибка 2: Считать count[x] = null как 0.**
В Dart `count[x] ?? 0` — явная обработка null.
