# Задача: Скалярное произведение разреженных векторов

> **Уровень сложности:** Medium  
> **Тема:** Два указателя, run-length encoding  
> **Встречается на:** Яндекс, Google

---

## 1. Что просят?

```
Векторы заданы как список пар (value, count).
Пара (v, c) означает "значение v повторяется c раз".

a = [(3,2),(0,1),(5,1)] → вектор [3,3,0,5]
b = [(2,1),(0,2),(4,1)] → вектор [2,0,0,4]

Скалярное произведение: 3*2 + 3*0 + 0*0 + 5*4 = 6+0+0+20 = 26
```

---

## 2. Ключевая идея

Два указателя — по одному на каждый вектор. Обрабатываем минимальный из двух текущих отрезков за один шаг. Не разворачиваем векторы в полные массивы.

---

## 3. Пошаговый алгоритм

```
i=0, j=0 (указатели на текущий run)
remA = a[0].count, remB = b[0].count (сколько осталось в текущем run)

Пока i < len(a) И j < len(b):
  step = min(remA, remB)
  result += a[i].value * b[j].value * step
  remA -= step; remB -= step
  если remA == 0: i++, remA = a[i].count
  если remB == 0: j++, remB = b[j].count

Вернуть result
```

---

## 4. Трассировка

**a=[(3,2),(0,1),(5,1)], b=[(2,1),(0,2),(4,1)]**

```
remA=2, remB=1

Шаг 1: step=min(2,1)=1, result += 3*2*1=6, remA=1, remB=0
  remB=0 → j=1, remB=2

Шаг 2: step=min(1,2)=1, result += 3*0*1=0, remA=0, remB=1
  remA=0 → i=1, remA=1

Шаг 3: step=min(1,1)=1, result += 0*0*1=0, remA=0, remB=0
  remA=0 → i=2, remA=1; remB=0 → j=2, remB=1

Шаг 4: step=min(1,1)=1, result += 5*4*1=20, remA=0, remB=0
  i=3 >= len(a) → стоп

result = 6+0+0+20 = 26 ✓
```

---

## 5. Сложность

| | Сложность |
|---|---|
| Время | O(n+m) по суммарной длине списков run |
| Память | O(1) |

---

## 6. Типичные ошибки

**Ошибка 1: Развернуть векторы в полные массивы — O(N) памяти.**
Это работает, но теряет преимущество кодирования.

**Ошибка 2: Неправильно вычислять step.**
`step = min(remA, remB)` — берём минимум, чтобы не выйти за границу меньшего run.

**Ошибка 3: Не проверять границы после продвижения указателя.**
После `i++` проверить `i < len(a)` перед обращением к `a[i]`.
