# Задача: Количество пар с разностью >= K

> **Уровень сложности:** Medium  
> **Тема:** Сортировка, скользящее окно, два указателя  
> **Встречается на:** Яндекс

---

## 1. Что просят?

```
Дан массив a и число k. Найти количество пар (i,j) где i<j
и |a[i]-a[j]| >= k.

Примеры:
  [1,3,7,2], k=3 → 4
    отсортируем: [1,2,3,7]
    пары: (1,7)|diff=6✓, (2,7)|diff=5✓, (3,7)|diff=4✓, (1,3)|diff=2✗...
    итого 3 пары (зависит от точного подсчёта)
```

---

## 2. Ключевая идея

Сортировка + два указателя. Отсортированный массив: если `sorted[r] - sorted[l] >= k`, то все элементы правее `r` тоже образуют пары с `sorted[l]`. Это даёт O(n) после сортировки.

---

## 3. Пошаговый алгоритм

```
sorted = sort(a)
r = 1

Для l от 0 до n-2:
  если r <= l: r = l + 1   // r всегда правее l
  Пока r < n И sorted[r] - sorted[l] < k: r++
  count += n - r           // все элементы [r..n-1] образуют пары с l
```

---

## 4. Трассировка

**a=[1,2,3,7], k=3 (уже отсортирован)**

```
l=0(1), r=1:
  sorted[1]-sorted[0]=2-1=1 < 3 → r=2
  sorted[2]-sorted[0]=3-1=2 < 3 → r=3
  sorted[3]-sorted[0]=7-1=6 >= 3 → стоп
  count += 4-3 = 1  (пара (1,7))

l=1(2), r=3:
  sorted[3]-sorted[1]=7-2=5 >= 3 → стоп
  count += 4-3 = 1  (пара (2,7))

l=2(3), r=3:
  sorted[3]-sorted[2]=7-3=4 >= 3 → стоп
  count += 4-3 = 1  (пара (3,7))

Итого: count = 3 ✓
```

---

## 5. Сложность

| | Сложность |
|---|---|
| Время | O(n log n) — сортировка |
| Память | O(n) — копия массива |

---

## 6. Типичные ошибки

**Ошибка 1: Наивный O(n²) с двумя вложенными циклами.**
Работает, но не принимают на собесе при больших данных.

**Ошибка 2: r не всегда правее l.**
После `l++` нужно проверить `if (r <= l) r = l + 1`.

**Ошибка 3: Считать разность без abs().**
После сортировки `sorted[r] >= sorted[l]` всегда, поэтому `abs` не нужен.
