# Задача: Фильтрация отсортированного массива по списку исключений

> **Уровень сложности:** Easy  
> **Тема:** Два указателя, отсортированные массивы  
> **Встречается на:** Яндекс

---

## 1. Что просят?

```
Даны два отсортированных массива a и exclude.
Вернуть все элементы a, которые не встречаются в exclude.

Примеры:
  a=[1,2,3,4,5], exclude=[2,4] → [1,3,5]
  a=[1,1,2,3],   exclude=[1,3] → [2]
```

---

## 2. Ключевая идея

Два указателя по отсортированным массивам. Для каждого элемента a[i] находим в exclude нужную позицию за O(1) (не с начала, а продолжая с предыдущей позиции).

---

## 3. Пошаговый алгоритм

```
i = 0, j = 0

Пока i < len(a):
  Двигаем j пока exclude[j] < a[i]
  Если j >= len(exclude) ИЛИ exclude[j] != a[i]:
    result.add(a[i])   // a[i] не в exclude
  i++
```

---

## 4. Трассировка

**a=[1,2,3,4,5], exclude=[2,4]**

```
i=0(1), j=0(2): exclude[0]=2 >= 1, 2≠1 → add(1)
i=1(2), j=0(2): exclude[0]=2 >= 2, 2==2 → skip
i=2(3), j=0(2): exclude[0]=2 < 3 → j=1(4); exclude[1]=4 >= 3, 4≠3 → add(3)
i=3(4), j=1(4): exclude[1]=4 >= 4, 4==4 → skip
i=4(5), j=1(4): exclude[1]=4 < 5 → j=2; j>=len → add(5)

Результат: [1,3,5] ✓
```

---

## 5. Сложность

| | Сложность |
|---|---|
| Время | O(n+m) |
| Память | O(n) — результат |

---

## 6. Типичные ошибки

**Ошибка 1: Использовать contains() для каждого элемента — O(n*m).**
Два указателя позволяют сделать O(n+m).

**Ошибка 2: Сбрасывать j на 0 для каждого нового элемента a.**
j продолжает движение с предыдущей позиции — это и даёт O(n+m).
