# Задача: Удалить нули из массива

> **Уровень сложности:** Easy  
> **Тема:** Два указателя, массивы  
> **Аналог на LeetCode:** 283. Move Zeroes  
> **Встречается на:** Яндекс, Facebook, Google

---

## 1. Что просят?

```
Удалить все нули из массива in-place, сохранив относительный порядок
ненулевых элементов. Нули переместить в конец.

Примеры:
  [0,1,0,3,12] → [1,3,12,0,0]
  [1,0,0,2,3]  → [1,2,3,0,0]
  [0,0,0]      → [0,0,0]
```

---

## 2. Ключевая идея

Указатель записи `writePos`. Один проход: когда встречаем ненулевой элемент — записываем его на `writePos` и двигаем `writePos` вперёд. В конце заполняем хвост нулями.

---

## 3. Два подхода

**Подход 1: Перезапись (2 прохода)**
```
Проход 1: для каждого nums[i] != 0: nums[writePos++] = nums[i]
Проход 2: пока writePos < len: nums[writePos++] = 0
```

**Подход 2: Swap (1 проход, меньше записей)**
```
writePos = 0
для каждого i где nums[i] != 0:
  swap(nums[writePos], nums[i])
  writePos++
```
Swap хорош тем, что не портит ненулевые элементы в хвосте — меняет их с нулями.

---

## 4. Трассировка

**Вход:** `[0,1,0,3,12]`

```
writePos=0

i=0: 0 → пропуск
i=1: 1 → nums[0]=1, writePos=1
i=2: 0 → пропуск
i=3: 3 → nums[1]=3, writePos=2
i=4: 12 → nums[2]=12, writePos=3

Заполняем хвост: nums[3]=0, nums[4]=0

Результат: [1,3,12,0,0] ✓
```

---

## 5. Сложность

| | Сложность |
|---|---|
| Время | O(n) |
| Память | O(1) |

---

## 6. Типичные ошибки

**Ошибка 1: Использовать remove() в цикле — O(n²).**
Каждый remove сдвигает весь массив. Используй два указателя.

**Ошибка 2: Не заполнять хвост нулями при подходе с перезаписью.**
После переноса ненулевых элементов нужно обнулить хвост.
