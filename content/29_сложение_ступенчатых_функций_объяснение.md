# Задача: Сложение ступенчатых функций

> **Уровень сложности:** Medium  
> **Тема:** Merge отсортированных списков, два указателя  
> **Встречается на:** Яндекс

---

## 1. Что просят?

```
Ступенчатая функция задаётся списком точек излома (x, y):
значение y начинается с x и длится до следующей точки.

f = [(0,1),(3,2),(7,0)]: f(0..2)=1, f(3..6)=2, f(7+)=0
g = [(0,2),(5,1)]:       g(0..4)=2, g(5+)=1

(f+g) = [(0,3),(3,4),(5,3),(7,1)]
```

---

## 2. Ключевая идея

Merge двух отсортированных списков точек. Для каждой точки излома (из f или g) пересчитываем сумму. Аналог слияния двух отсортированных массивов.

---

## 3. Пошаговый алгоритм

```
i=0 (указатель по f), j=0 (по g)
fVal=0, gVal=0 (текущие значения)

Пока i < len(f) ИЛИ j < len(g):
  nextX = min(f[i].x, g[j].x)  (с проверкой границ)
  если f[i].x == nextX: fVal = f[i].y, i++
  если g[j].x == nextX: gVal = g[j].y, j++
  если sum = fVal+gVal отличается от последнего в result:
    result.add((nextX, sum))
```

---

## 4. Трассировка

**f=[(0,1),(3,2),(7,0)], g=[(0,2),(5,1)]**

```
i=0(0,1), j=0(0,2): nextX=0
  f[0].x==0→fVal=1,i=1; g[0].x==0→gVal=2,j=1; sum=3 → add(0,3)

i=1(3,2), j=1(5,1): nextX=3
  f[1].x==3→fVal=2,i=2; sum=2+2=4 → add(3,4)

i=2(7,0), j=1(5,1): nextX=5
  g[1].x==5→gVal=1,j=2; sum=2+1=3 → add(5,3)

i=2(7,0), j≥len(g): nextX=7
  f[2].x==7→fVal=0,i=3; sum=0+1=1 → add(7,1)

result=[(0,3),(3,4),(5,3),(7,1)] ✓
```

---

## 5. Сложность

| | Сложность |
|---|---|
| Время | O(n+m) |
| Память | O(n+m) |

---

## 6. Типичные ошибки

**Ошибка 1: Обновить только одну функцию когда точки совпадают.**
Если f[i].x == g[j].x — обновить обе. Проверять через два отдельных `if`, не `if-else`.

**Ошибка 2: Не убирать точки с одинаковым значением.**
Если два соседних сегмента имеют одинаковую сумму — одна из точек лишняя.
