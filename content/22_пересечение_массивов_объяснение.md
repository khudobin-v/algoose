# Задача: Пересечение двух отсортированных массивов

> **Уровень сложности:** Easy  
> **Тема:** Два указателя, отсортированные массивы  
> **Аналог на LeetCode:** 350. Intersection of Two Arrays II  
> **Встречается на:** Яндекс, Google, Amazon

---

## 1. Что просят?

```
Даны два отсортированных массива. Найти их пересечение
(с учётом кратности — каждый элемент включается min(count_a, count_b) раз).

Примеры:
  [1,2,2,3] ∩ [2,2,4] = [2,2]
  [1,3,5]   ∩ [2,4,6] = []
  [1,2,3]   ∩ [1,2,3] = [1,2,3]
```

---

## 2. Ключевая идея

Два указателя. Оба массива отсортированы — это позволяет за O(n+m) найти пересечение:
- если a[i] == b[j] → совпадение, добавляем, двигаем оба
- если a[i] < b[j] → a[i] точно нет в b, двигаем i
- если a[i] > b[j] → b[j] точно нет в a, двигаем j

---

## 3. Пошаговый алгоритм

```
i = 0, j = 0, result = []

Пока i < len(a) И j < len(b):
  если a[i] == b[j]: result.add(a[i]), i++, j++
  если a[i] < b[j]:  i++
  если a[i] > b[j]:  j++

Вернуть result
```

---

## 4. Трассировка

**Вход:** a=[1,2,2,3], b=[2,2,4]

```
i=0(1), j=0(2): 1<2 → i=1
i=1(2), j=0(2): 2==2 → result=[2], i=2, j=1
i=2(2), j=1(2): 2==2 → result=[2,2], i=3, j=2
i=3(3), j=2(4): 3<4 → i=4
i=4 >= len(a)=4 → стоп

Ответ: [2,2] ✓
```

---

## 5. Сложность

| | Сложность |
|---|---|
| Время | O(n+m) |
| Память | O(min(n,m)) — размер результата |

---

## 6. Типичные ошибки

**Ошибка 1: Использовать вложенные циклы — O(n*m).**
Два указателя работают за O(n+m) благодаря сортировке.

**Ошибка 2: Не учесть дубликаты.**
Если a=[2,2] и b=[2,2], то пересечение [2,2] — оба элемента, не один.
Два указателя автоматически это обрабатывают.
