# Задача: Оптимизация пути — удаление петель

> **Уровень сложности:** Medium  
> **Тема:** HashMap, симуляция  
> **Встречается на:** Яндекс

---

## 1. Что просят?

```
Дан путь из символов L/R/U/D. Упростить его, убрав петли
(участки пути которые возвращают в уже посещённую точку).

Примеры:
  "RDLU" → ""   (путь образует квадрат и возвращается в (0,0))
  "RRLL" → ""   (идёт вправо-вправо-влево-влево)
  "RRLD" → "RD" (R→R→L убирает петлю, остаётся сдвиг вправо и вниз)
```

---

## 2. Ключевая идея

HashMap координат → индекс в текущем пути. При повторном посещении точки — удаляем все шаги от первого посещения до текущего (это и есть петля). Также чистим все промежуточные точки из HashMap.

---

## 3. Пошаговый алгоритм

```
visited = {(0,0): 0}
steps = []
(x, y) = (0, 0)

Для каждого направления dir:
  двигаемся: (x,y) += delta[dir]
  steps.add(dir)
  
  если (x,y) в visited:
    loopStart = visited[(x,y)]
    steps = steps[0..loopStart]  // убираем петлю
    удалить из visited все с index >= loopStart
  иначе:
    visited[(x,y)] = len(steps)
```

---

## 4. Трассировка

**"RDLU"**

```
(0,0): visited={(0,0):0}, steps=[]

'R': (1,0), steps=["R"], visited={(0,0):0, (1,0):1}
'D': (1,-1), steps=["R","D"], visited={(0,0):0,(1,0):1,(1,-1):2}
'L': (0,-1), steps=["R","D","L"], visited+={(0,-1):3}
'U': (0,0) — уже в visited! loopStart=0
  steps = steps[0..0] = []
  visited = {}
  
Ответ: "" ✓
```

## 6. Типичные ошибки

**Ошибка 1: Не очищать промежуточные точки из visited при удалении петли.**
Если не очистить — после возврата в точку алгоритм считает что промежуточные точки всё ещё "в пути".

**Ошибка 2: Неправильное отображение индексов.**
`visited[pos] = steps.length` сохраняет индекс ПОСЛЕ добавления шага — нужно быть внимательным с off-by-one.
