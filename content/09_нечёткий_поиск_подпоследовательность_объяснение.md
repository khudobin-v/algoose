# Задача: Нечёткий поиск — проверка подпоследовательности

> **Уровень сложности:** Easy  
> **Тема:** Два указателя, строки  
> **Аналог на LeetCode:** 392. Is Subsequence  
> **Встречается на:** Яндекс, Google, Amazon

---

## 1. Что просят?

```
Даны строки s и t. Проверить, является ли s подпоследовательностью t.
s является подпоследовательностью t если все символы s встречаются в t
в том же порядке слева направо, но не обязательно подряд.

Примеры:
  isSubsequence("ace", "abcde") → true   (a..c..e — все по порядку)
  isSubsequence("aec", "abcde") → false  (e стоит ПОСЛЕ c, но в s — раньше)
  isSubsequence("", "abc")      → true   (пустая строка — всегда подпоследовательность)
  isSubsequence("abc", "")      → false
```

---

## 2. Ключевая идея

Два указателя: один по s, другой по t.
Проходим t слева направо. Когда текущий символ t совпадает с текущим символом s — сдвигаем указатель по s.
Если указатель по s дошёл до конца — все символы найдены.

---

## 3. Пошаговый алгоритм

```
i = 0  (указатель по s)
j = 0  (указатель по t)

Пока i < len(s) И j < len(t):
  если s[i] == t[j]: i++   // нашли очередной символ s
  j++                       // t просматриваем всегда

Ответ: i == len(s)          // все символы s найдены?
```

---

## 4. Трассировка

**Вход:** s="ace", t="abcde"

```
i=0('a'), j=0('a'): совпало → i=1, j=1
i=1('c'), j=1('b'): нет → j=2
i=1('c'), j=2('c'): совпало → i=2, j=3
i=2('e'), j=3('d'): нет → j=4
i=2('e'), j=4('e'): совпало → i=3, j=5

i=3 == len("ace")=3 → true ✓
```

**Вход:** s="aec", t="abcde"

```
i=0('a'), j=0('a'): совпало → i=1, j=1
i=1('e'), j=1('b'): нет → j=2
i=1('e'), j=2('c'): нет → j=3
i=1('e'), j=3('d'): нет → j=4
i=1('e'), j=4('e'): совпало → i=2, j=5
Цикл: j=5 >= len(t)=5, i=2 < len(s)=3

i=2 != 3 → false ✓
```

## 6. Типичные ошибки

**Ошибка 1: Забыть граничный случай пустой строки.**
```dart
// Пустая s — всегда подпоследовательность
// Алгоритм обрабатывает это автоматически: i=0=len(s) → return true
```

**Ошибка 2: Двигать j только когда символы совпадают.**
```dart
// Плохо — зациклится если символы не совпадают
if (s[i] == t[j]) { i++; j++; }
// Хорошо — j двигаем всегда
if (s[i] == t[j]) i++;
j++;
```

**Ошибка 3: Проверять i == len(s) ДО конца цикла.**
Нужно проверять после — цикл сам завершится когда i достигнет конца.

---

## 7. Псевдокод

```
функция isSubsequence(s, t):
  i = 0
  для каждого символа c в t:
    если i < len(s) и s[i] == c:
      i++
  вернуть i == len(s)
```
