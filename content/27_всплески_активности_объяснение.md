# Задача: Всплески активности — события за последние 5 минут

> **Уровень сложности:** Easy-Medium  
> **Тема:** Скользящее окно, очередь  
> **Аналог на LeetCode:** 933. Number of Recent Calls  
> **Встречается на:** Яндекс, Google

---

## 1. Что просят?

```
Дана последовательность событий с временными метками (возрастающий порядок).
Для каждого i-го события посчитать, сколько событий произошло
за последние 5 минут (300 секунд), включая текущее.

Пример:
  timestamps = [10, 60, 120, 300, 310]
  
  t=10:  окно [0..10]   → событий: 1  (только t=10)
  t=60:  окно [0..60]   → событий: 2  (t=10, t=60)
  t=120: окно [0..120]  → событий: 3  (t=10, t=60, t=120)
  t=300: окно [0..300]  → событий: 4  (все четыре)
  t=310: окно [10..310] → событий: 4  (t=60,t=120,t=300,t=310 — t=10 выпало)
```

---

## 2. Ключевая идея

Скользящее окно на очереди. Добавляем новое событие в конец очереди.
Удаляем из начала события старше 300 секунд. Размер очереди = ответ.

---

## 3. Пошаговый алгоритм

```
window = пустая очередь

Для каждого timestamp t:
  пока window не пуст И t - window.first > 300:
    window.removeFirst()
  window.addLast(t)
  result.add(window.size)
```

---

## 4. Трассировка

**timestamps=[10,60,120,300,310]**

```
t=10:  window=[], 10-? > 300 нет → add(10), window=[10], size=1
t=60:  10-60=-50 нет → add(60), window=[10,60], size=2
t=120: ...add(120), window=[10,60,120], size=3
t=300: ...add(300), window=[10,60,120,300], size=4
t=310: 310-10=300 > 300? нет (не строго) → add(310), window=[10,60,120,300,310]
       ИЛИ если условие >=300: 310-10=300 → удаляем 10 → size=4
```

> Уточни у интервьюера: граница включена или нет (t - timestamp > 300 или >= 300).

---

## 5. Сложность

| | Сложность |
|---|---|
| Время | O(n) суммарно — каждое событие добавляется и удаляется раз |
| Память | O(W) — размер окна |

---

## 6. Типичные ошибки

**Ошибка 1: Использовать список и removeAt(0) — O(n) на удаление.**
Использовать `Queue` (двусторонняя очередь) с `removeFirst()` — O(1).

**Ошибка 2: Удалять не из начала, а из конца.**
Старые события всегда в начале (очередь FIFO), удалять надо `removeFirst()`.

**Ошибка 3: Не включить текущее событие в окно перед подсчётом.**
Сначала добавить, потом считать размер.
