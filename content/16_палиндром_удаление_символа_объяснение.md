# Задача: Палиндром с удалением одного символа

> **Уровень сложности:** Easy-Medium  
> **Тема:** Два указателя, строки  
> **Аналог на LeetCode:** 680. Valid Palindrome II  
> **Встречается на:** Яндекс, Facebook, Amazon

---

## 1. Что просят?

```
Дана строка. Проверить, можно ли удалить не более одного символа,
чтобы строка стала палиндромом.

Примеры:
  "abca"    → true  (удаляем 'b': "aca" или удаляем 'c': "aba")
  "racecar" → true  (уже палиндром)
  "abcd"    → false (нужно удалить больше одного символа)
```

---

## 2. Ключевая идея

Два указателя с краёв. Когда находим несовпадение — пробуем пропустить левый или правый символ.
Если хотя бы один вариант даёт палиндром — ответ `true`.

---

## 3. Пошаговый алгоритм

```
left = 0, right = len(s) - 1

Пока left < right:
  если s[left] == s[right]: left++, right--
  иначе:
    вернуть isPalindrome(s, left+1, right) ИЛИ isPalindrome(s, left, right-1)

Вернуть true (строка уже палиндром)
```

---

## 4. Трассировка

**Вход:** `"abca"`

```
left=0('a'), right=3('a'): совпало → left=1, right=2
left=1('b'), right=2('c'): НЕ совпало!
  Вариант 1: isPalindrome("abca", 2, 2) → "c" → true ✓
  → return true
```

**Вход:** `"abcd"`

```
left=0('a'), right=3('d'): НЕ совпало!
  Вариант 1: isPalindrome("abcd", 1, 3) → "bcd" → 'b'≠'d' → false
  Вариант 2: isPalindrome("abcd", 0, 2) → "abc" → 'a'≠'c' → false
  → return false ✓
```

## 6. Типичные ошибки

**Ошибка 1: Рекурсивно вызывать основную функцию.**
```dart
// Плохо — разрешит удалять НЕСКОЛЬКО символов
return canBePalindrome(s.substring(left+1, right+1)) ||
       canBePalindrome(s.substring(left, right));

// Хорошо — вспомогательная функция без права на ошибку
return _isPalindrome(s, left+1, right) || _isPalindrome(s, left, right-1);
```

**Ошибка 2: Не обрабатывать случай когда строка уже палиндром.**
Если цикл завершился без несовпадений — строка палиндром, удаление допустимо → `true`.
