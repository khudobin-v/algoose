# Задача: Разбиение массива на три части минимальной стоимости

> **Уровень сложности:** Medium  
> **Тема:** Жадный алгоритм, один проход  
> **Встречается на:** Яндекс

---

## 1. Что просят?

```
Разбить массив на 3 непустых непрерывных части.
Стоимость разбиения = сумма первых элементов всех трёх частей.
Минимизировать стоимость.

Первая часть всегда начинается с a[0].
Нужно выбрать split1 (начало 2-й части) и split2 (начало 3-й части).

Пример:
  a=[1,4,2,5,3], стоимость = a[0] + a[split1] + a[split2]
  = 1 + min(a[1..3]) + соответствующий min(a[split1+1..4])
```

---

## 2. Ключевая идея

Стоимость = `a[0] + a[split1] + a[split2]`, где `1 <= split1 < split2 <= n-1`.
`a[0]` фиксировано. Нужно минимизировать `a[split1] + a[split2]`.

Один проход: для каждого `split2` берём лучший (минимальный) `split1` среди всех `a[1..split2-1]`.
Поддерживаем текущий минимум `min2` — это и есть лучший `a[split1]`.

---

## 3. Пошаговый алгоритм

```
min2 = a[1], bestSplit1 = 1
bestCost = a[0] + a[1] + a[2]

Для j от 2 до n-1:
  cost = a[0] + min2 + a[j]
  если cost < bestCost: обновляем bestCost, bestI=bestSplit1, bestJ=j
  если a[j] < min2: min2=a[j], bestSplit1=j  // обновляем минимум для след. j
```

---

## 4. Сложность

| | Сложность |
|---|---|
| Время | O(n) |
| Память | O(1) |

---

## 5. Типичные ошибки

**Ошибка 1: Обновлять min2 ДО проверки cost.**
Нужно сначала проверить `cost = a[0] + min2 + a[j]`, потом обновить `min2`.
Иначе `split1 >= split2` — нарушение условия.

**Ошибка 2: Не запоминать индекс лучшего split1.**
Для восстановления разбиения нужно хранить не только минимальное значение, но и его индекс.
