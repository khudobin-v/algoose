# Задача: Проверка строк на одно исправление (Edit Distance 1)

> **Уровень сложности:** Easy-Medium  
> **Тема:** Строки, два указателя  
> **Аналог на LeetCode:** 161. One Edit Distance  
> **Встречается на:** Яндекс, Google, Microsoft

---

## 1. Что просят?

```
Даны две строки. Проверить, можно ли одну получить из другой
не более чем одним исправлением.

Одно исправление — это:
  - Замена одного символа
  - Вставка одного символа
  - Удаление одного символа

Примеры:
  ("cat", "dog")  →  false   (нужно 3 замены)
  ("cat", "cats") →  true    (вставка 's')
  ("cat", "cut")  →  true    (замена 'a' → 'u')
  ("cat", "cat")  →  true    (0 исправлений — тоже ок? уточни у интервьюера!)
  ("cats", "cat") →  true    (удаление 's')
  ("acts", "cat") →  false   (нужно 2 операции)
```

> ⚠️ Уточни: "не более одного" или "ровно одного" исправления?  
> В задаче Яндекса обычно "не более одного" (т.е. одинаковые строки тоже true).

---

## 2. Как думать на собесе

### Шаг 1: Определи случаи по длинам строк
```
len(s) == len(t): единственная возможная операция — ЗАМЕНА
  → идём слева направо, считаем отличия, должно быть ≤ 1

|len(s) - len(t)| == 1: единственная возможная операция — ВСТАВКА/УДАЛЕНИЕ
  → идём по более длинной строке, пропускаем один символ

|len(s) - len(t)| > 1: гарантированно false
```

### Шаг 2: Алгоритм

**Случай 1: Одинаковая длина (замена)**
```
Сравниваем посимвольно. Если разных символов > 1 — false.
```

**Случай 2: Разная длина на 1 (вставка/удаление)**
```
Ставим два указателя — i для s, j для t.
Идём вперёд. Если символы совпадают — двигаем оба.
Если не совпадают — разрешаем один пропуск в более длинной строке.
Если пропуск уже был — false.
```

## 3. Код на Dart

```dart
bool isOneEditAway(String s, String t) {
  final lenS = s.length;
  final lenT = t.length;

  // Разница длин больше 1 — точно false
  if ((lenS - lenT).abs() > 1) return false;

  // Убедимся что s — более короткая строка
  if (lenS > lenT) return isOneEditAway(t, s);

  // Теперь lenT >= lenS
  int i = 0; // указатель по s (короткой)
  int j = 0; // указатель по t (длинной или равной)
  bool usedEdit = false;

  while (i < lenS && j < lenT) {
    if (s[i] != t[j]) {
      if (usedEdit) return false; // второй раз не совпало
      usedEdit = true;

      if (lenS == lenT) {
        // Замена: двигаем оба
        i++;
        j++;
      } else {
        // Вставка/удаление: двигаем только длинную
        j++;
      }
    } else {
      i++;
      j++;
    }
  }

  return true;
}
```

## 4. Трассировка

### Замена: ("cat", "cut")
```
lenS=3, lenT=3 → случай замены

i=0,j=0: 'c'=='c' → i=1,j=1
i=1,j=1: 'a'≠'u' → usedEdit=true, i=2,j=2 (замена)
i=2,j=2: 't'=='t' → i=3,j=3
Конец: return true ✓
```

### Вставка: ("cat", "cats")
```
lenS=3, lenT=4 → случай вставки

i=0,j=0: 'c'=='c' → i=1,j=1
i=1,j=1: 'a'=='a' → i=2,j=2
i=2,j=2: 't'=='t' → i=3,j=3
Цикл закончился (i=lenS=3)
Остался хвост в t — 's' — это ровно 1 лишний символ
return true ✓
```

### Два различия: ("acts", "cat")
```
lenS=3 (cat), lenT=4 (acts) → случай вставки (swap → s="cat", t="acts")

i=0,j=0: 'c'≠'a' → usedEdit=true, j=1 (двигаем длинную)
i=0,j=1: 'c'≠'c' — подождём: 'c'=='c' → i=1,j=2
i=1,j=2: 'a'≠'t' → usedEdit уже true → return false ✓
```

## 5. Частые ошибки

**Ошибка 1: Забыть случай когда хвост остался.**  
```
("cat", "cats"): после цикла i=3=lenS, j=3<lenT — остался 's'.
Это валидно! Код автоматически вернёт true потому что цикл завершился нормально.
```

**Ошибка 2: Не свапнуть строки чтобы s была короче.**  
Код должен работать одинаково для ("cat","cats") и ("cats","cat").

**Ошибка 3: Сравнивать длины как int без abs().**
```dart
if (lenS - lenT > 1) // Плохо! Если lenS < lenT, это всегда false
if ((lenS - lenT).abs() > 1) // Хорошо
```
