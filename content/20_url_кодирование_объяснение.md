# Задача: URL-кодирование строки

> **Уровень сложности:** Easy  
> **Тема:** Строки, два прохода  
> **Встречается на:** Яндекс, Microsoft (классика)

---

## 1. Что просят?

```
Заменить все пробелы в строке на "%20" — in-place, без лишней памяти.

Примеры:
  "Mr John Smith" → "Mr%20John%20Smith"
  "hello world"   → "hello%20world"
```

---

## 2. Ключевая идея

Два прохода:
1. Считаем количество пробелов → знаем итоговую длину
2. Идём с КОНЦА строки, записываем символы с конца итогового массива

Движение с конца гарантирует, что мы не затрём ещё не обработанные символы.

---

## 3. Пошаговый алгоритм

```
Проход 1: count = кол-во пробелов
newLen = len(s) + count * 2  (каждый пробел → 3 символа вместо 1)

Проход 2: writePos = newLen - 1
Идём i от конца s до 0:
  если s[i] == ' ':
    result[writePos]   = '0'
    result[writePos-1] = '2'
    result[writePos-2] = '%'
    writePos -= 3
  иначе:
    result[writePos] = s[i]
    writePos--
```

---

## 4. Трассировка

**Вход:** `"a b"` (3 символа, 1 пробел → newLen=5)

```
writePos = 4

i=2('b'): result[4]='b', writePos=3
i=1(' '): result[3]='0', result[2]='2', result[1]='%', writePos=0
i=0('a'): result[0]='a', writePos=-1

result = "a%20b" ✓
```

---

## 5. Сложность

| | Сложность |
|---|---|
| Время | O(n) |
| Память | O(1) дополнительной (не считая результата) |

---

## 6. Типичные ошибки

**Ошибка 1: Идти с начала — затрём ещё не обработанные символы.**
Всегда идти с конца при расширении строки in-place.

**Ошибка 2: Не учесть, что '%20' занимает 3 символа.**
`newLen = s.length + spaceCount * 2` (добавляем 2, а не 3, потому что сам пробел уже занимает 1 место).
