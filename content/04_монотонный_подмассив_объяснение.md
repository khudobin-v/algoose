# Задача: Максимальный монотонный подотрезок

> **Уровень сложности:** Easy-Medium  
> **Тема:** Массивы, два указателя, один проход  
> **Встречается на:** Яндекс (разминочная задача)

---

## 1. Что просят?

```
Дан массив чисел a_1, a_2, ..., a_n.
Найти строго монотонный подотрезок (строго возрастающий ИЛИ строго убывающий)
максимальной длины. Вернуть пару индексов {begin, end}.

Примеры:
  [2, 7, 5, 4, 3]  →  {1, 3}   // подотрезок [7,5,4,3] длиной 4
  [1, 1]           →  {0, 1} или {1, 1}  // любой одиночный элемент
  [1, 2, 3, 4]     →  {0, 3}   // весь массив возрастающий
```

---

## 2. Как думать на собесе

### Шаг 1: Уточни "строго"
> "Строго монотонный — это значит никаких равных соседних элементов?
> Например [1,1,2] — это не монотонный?"  
> Да, строго. [1,1] → любой одноэлементный отрезок.

### Шаг 2: Алгоритм
Один проход. Отслеживаем начало текущего монотонного отрезка и его направление.

```
start = 0
direction = NONE  (не определено)

Идём i от 1 до n-1:
  Смотрим на пару (a[i-1], a[i]):
    если a[i] > a[i-1]: текущее направление = ВВЕРХ
    если a[i] < a[i-1]: текущее направление = ВНИЗ
    если a[i] == a[i-1]: разрыв — начинаем новый отрезок с i
  
  Если направление сменилось (ВВЕРХ→ВНИЗ или наоборот):
    разрыв — начинаем новый отрезок с i-1
  
  Обновляем лучший отрезок если текущий длиннее
```

## 3. Код на Dart

```dart
(int, int) longestMonotone(List<int> a) {
  if (a.length <= 1) return (0, a.length - 1);

  int bestStart = 0;
  int bestEnd = 0;
  int start = 0;
  
  // 1 = возрастание, -1 = убывание, 0 = не определено
  int direction = 0;

  for (int i = 1; i < a.length; i++) {
    int curDir;
    if (a[i] > a[i - 1]) {
      curDir = 1;
    } else if (a[i] < a[i - 1]) {
      curDir = -1;
    } else {
      // Равные — разрыв
      start = i;
      direction = 0;
      continue;
    }

    if (direction != 0 && curDir != direction) {
      // Смена направления — начинаем новый отрезок
      start = i - 1;
    }
    
    direction = curDir;

    // Обновляем лучший результат
    if (i - start > bestEnd - bestStart) {
      bestStart = start;
      bestEnd = i;
    }
  }

  return (bestStart, bestEnd);
}
```

## 4. Трассировка

**Вход:** `[2, 7, 5, 4, 3]`

```
start=0, direction=0, best=(0,0)

i=1: a[1]=7 > a[0]=2 → curDir=1, direction=0→1, best=(0,1)
i=2: a[2]=5 < a[1]=7 → curDir=-1, direction=1≠-1 → start=1, direction=-1, best=(0,1) (длина 2=2, не лучше)
i=3: a[3]=4 < a[2]=5 → curDir=-1, direction=-1==-1, i-start=2 > 1 → best=(1,3)
i=4: a[4]=3 < a[3]=4 → curDir=-1, direction=-1==-1, i-start=3 > 2 → best=(1,4)

Ответ: (1, 4) = [7,5,4,3] длиной 4 ✓
```

---

## 5. Частые ошибки

**Ошибка 1: Не обработать равные элементы.**  
[1,1,2] — это НЕ монотонный. Равные элементы разрывают отрезок.

**Ошибка 2: Смену направления начинать с i, а не с i-1.**  
```
[1,2,1]: на i=2 видим спуск. Новый отрезок начинается с a[1]=2, то есть start=i-1=1.
Если написать start=i — потеряем элемент 2 из нового отрезка.
```

**Ошибка 3: Забыть обновить best в конце итерации.**  
Самый длинный отрезок может заканчиваться в последнем элементе.
