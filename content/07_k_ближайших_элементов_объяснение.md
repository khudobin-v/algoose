# Задача: Найти K ближайших к заданному элементов в отсортированном массиве

> **Уровень сложности:** Medium  
> **Тема:** Два указателя, бинарный поиск  
> **Аналог на LeetCode:** 658. Find K Closest Elements  
> **Встречается на:** Яндекс, Google, Amazon

---

## 1. Читаем условие — что реально просят?

```
Дан отсортированный по неубыванию массив целых чисел a,
индекс элемента index и целое число k.

Вернуть k чисел из массива, ближайших по значению к a[index].
Порядок в ответе — любой.
```

### Примеры:
```
a=[2,3,5,7,11],  index=3, k=2  →  {5, 7}      // a[3]=7, берём 7 и 5
a=[4,12,15,15,24], index=1, k=3 →  {12,15,15}  // a[1]=12, берём 12,15,15
a=[2,3,5,7,11],  index=2, k=2  →  {3,5} или {5,7}  // a[2]=5, ничья — оба варианта ок
```

### Что значит "ближайший"?
Расстояние между элементами = |a[i] - a[index]|.  
Сам элемент `a[index]` имеет расстояние 0 — он всегда входит в ответ.

---

## 2. Как думать на собесе — пошагово

### Шаг 1: Переформулируй задачу

> "У меня есть отсортированный массив и опорный элемент. Нужно выбрать k элементов, ближайших к нему по значению. Опорный элемент всегда войдёт в ответ — его расстояние равно нулю."

---

### Шаг 2: Задай уточняющие вопросы

> "Что делать при ничьей? Например, a[index]=5, и слева 3 (расстояние 2), и справа 7 (расстояние 2) — кого брать?"

Из задачи: "вернуть в любом порядке" — при ничьей любой вариант допустим. Но уточни — на некоторых собесах добавляют правило "предпочитать меньший элемент".

---

### Шаг 3: Найди ключевое наблюдение

Массив **отсортирован**. Значит ближайшие к `a[index]` элементы находятся **рядом с ним** в массиве — слева и справа от `index`.

Визуально:
```
a = [2,  3,  5,  7,  11]
              ↑
           index=2, a[2]=5

Расстояния от 5:
  2  →  |2-5| = 3
  3  →  |3-5| = 2   ← ближе
  5  →  |5-5| = 0   ← сам элемент
  7  →  |7-5| = 2   ← ближе
  11 →  |11-5| = 6
```

Значит нужно "расширяться" от `index` влево и вправо, каждый раз беря более близкий элемент — как молния, расходящаяся от центра.

---

### Шаг 4: Придумай алгоритм — два указателя

```
Ставим:
  left  = index - 1   (первый кандидат слева)
  right = index + 1   (первый кандидат справа)

result = [a[index]]   // опорный элемент всегда берём

Пока не набрали k элементов:
  Сравниваем расстояния от a[index]:
    - |a[left]  - a[index]|  — расстояние до левого кандидата
    - |a[right] - a[index]|  — расстояние до правого кандидата
  Берём ближайший, двигаем его указатель внутрь
```

**Сложность: O(k)** — именно этого ждут на Яндексе.

---

## 3. Финальный код на Dart

```dart
List<int> findKClosest(List<int> a, int index, int k) {
  final target = a[index];
  final result = <int>[target];  // опорный элемент всегда в ответе

  int left = index - 1;
  int right = index + 1;

  while (result.length < k) {
    final leftDist = left >= 0 ? (a[left] - target).abs() : double.maxFinite;
    final rightDist = right < a.length ? (a[right] - target).abs() : double.maxFinite;

    if (leftDist <= rightDist) {
      result.add(a[left]);
      left--;
    } else {
      result.add(a[right]);
      right++;
    }
  }

  return result;
}
```

## 4. Трассировка на примере вручную

**Вход:** `a=[2,3,5,7,11]`, `index=3`, `k=2`

```
target = a[3] = 7
result = [7]
left = 2,  right = 4

Итерация 1: result.length=1 < k=2
  leftDist  = |a[2] - 7| = |5 - 7| = 2
  rightDist = |a[4] - 7| = |11 - 7| = 4
  2 < 4 → берём левый: result = [7, 5], left = 1

result.length=2 == k=2 → стоп

Ответ: [7, 5] ✓
```

---

**Вход:** `a=[4,12,15,15,24]`, `index=1`, `k=3`

```
target = a[1] = 12
result = [12]
left = 0,  right = 2

Итерация 1:
  leftDist  = |a[0] - 12| = |4 - 12| = 8
  rightDist = |a[2] - 12| = |15 - 12| = 3
  3 < 8 → берём правый: result = [12, 15], right = 3

Итерация 2:
  leftDist  = |a[0] - 12| = 8
  rightDist = |a[3] - 12| = |15 - 12| = 3
  3 < 8 → берём правый: result = [12, 15, 15], right = 4

result.length=3 == k=3 → стоп

Ответ: [12, 15, 15] ✓
```

## 5. Граничные случаи

```dart
// k=0 → пустой массив
findKClosest([1,2,3], 1, 0) → []

// k=1 → только сам элемент
findKClosest([1,2,3], 1, 1) → [2]

// k=N → весь массив
findKClosest([1,2,3], 1, 3) → [2,1,3] (в любом порядке)

// index в начале массива — left уйдёт за границу
findKClosest([1,2,3,4,5], 0, 3) → [1,2,3]
// здесь left=-1 сразу, leftDist=infinity → всегда берём правый

// index в конце массива — right уйдёт за границу
findKClosest([1,2,3,4,5], 4, 3) → [5,4,3]
```

## 6. Частые ошибки на собесе

**Ошибка 1: Выход за границы массива.**
```dart
// Плохо — упадёт если left < 0
final leftDist = (a[left] - target).abs();

// Хорошо — защищаемся
final leftDist = left >= 0 ? (a[left] - target).abs() : double.maxFinite;
```

**Ошибка 2: Не включить `a[index]` в результат.**  
Сам элемент — всегда ближайший (расстояние 0). Если забыть — результат будет неверным.

**Ошибка 3: Пытаться решить за O(n log n) через сортировку по расстоянию.**
```dart
// Так делать НЕ надо — O(n log n) и теряешь важное свойство
a.sort((x, y) => (x - target).abs().compareTo((y - target).abs()));
return a.take(k).toList();
```
На Яндексе ждут O(k) — два указателя. Если предложишь сортировку, спросят "а быстрее можно?"

**Ошибка 4: Считать что `double.maxFinite` нельзя сравнивать с int.**  
В Dart это работает корректно — `int` автоматически приводится к `num` при сравнении.

## 8. Сложность

| Операция      | Сложность |
|---------------|-----------|
| Время         | **O(k)**  |
| Память        | O(k)      |

Решение за O(n log n) через сортировку — не принимается. O(n) через линейный поход — допустимо, но не оптимально. Яндекс ждёт O(k).

## 9. Follow-up вопросы от Яндекса

**"А если массив не отсортирован?"**  
→ Тогда O(n) — нельзя использовать свойство сортировки. Можно через кучу (heap) за O(n log k).

**"А если index не дан — дано само значение X (которого может не быть в массиве)?"**  
→ Сначала бинарным поиском находим позицию вставки X, потом тот же алгоритм двух указателей. Это усложнённая версия — LeetCode 658.

**"А если нужно вернуть результат отсортированным?"**  
→ После алгоритма делаем `.sort()` за O(k log k). Или сразу собираем в правильном порядке — идём слева направо по окну [left+1 .. right-1].
