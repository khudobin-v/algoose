# Задача: Свёртка списка идентификаторов в интервалы

> **Уровень сложности:** Easy-Medium  
> **Тема:** Сортировка, жадный алгоритм, работа со строками  
> **Аналог на LeetCode:** 228. Summary Ranges  
> **Встречается на:** Яндекс, и многих других

---

## 1. Читаем условие — что реально просят?

```
Дан список целых неотрицательных чисел без повторений.
Нужно вернуть строку, где последовательные числа (отличающиеся на 1)
объединены в интервал вида "start-end".
Одиночные числа выводятся как есть.
Результат отсортирован по возрастанию.
```

### Примеры:
```
[1, 4, 5, 2, 3, 9, 8, 11, 0]  →  "0-5,8-9,11"
[10, 30, 50, 20]               →  "10,20,30,50"
[3, 1, 2, 4]                   →  "1-4"    ← все четыре подряд!
[]                             →  ""
[5]                            →  "5"
```

---

## 2. Как думать на собесе — пошагово

### Шаг 1: Переформулируй задачу своими словами

Прежде чем писать код — скажи вслух что понял:

> "Мне нужно отсортировать числа, потом пройти по ним и группировать подряд идущие в интервалы. Если разрыв между соседними числами больше 1 — закрываем текущий интервал и начинаем новый."

Это показывает интервьюеру что ты понял задачу правильно.

---

### Шаг 2: Уточни граничные случаи

Сразу спроси или проговори вслух:

> "Проверю пограничные случаи: пустой список → пустая строка, один элемент → просто это число, два соседних числа → интервал или два отдельных числа?"

Пример который ловит ошибки:
```
[3, 1, 2, 4]  →  "1-4"  (не "1-3,4" — все четыре идут подряд!)
```

---

### Шаг 3: Придумай алгоритм

Думай вслух:

> "Первое что нужно — отсортировать. Иначе невозможно найти последовательности.
> После сортировки иду слева направо и отслеживаю начало текущего интервала.
> Если следующий элемент = текущий + 1, значит интервал продолжается.
> Если нет — закрываю интервал и начинаю новый."

**Визуально:**
```
Вход: [1, 4, 5, 2, 3, 9, 8, 11, 0]

После сортировки: [0, 1, 2, 3, 4, 5, 8, 9, 11]
                   ↑___________↑  ↑___↑  ↑
                   интервал 0-5  интервал 8-9  одиночка 11
```

---

### Шаг 4: Определи структуру данных

Нужно отслеживать всего две вещи:
- `start` — начало текущего интервала
- `end` — конец текущего интервала (он же — текущий элемент)

Когда разрыв — сохраняем `[start, end]` и делаем `start = следующий элемент`.

## 3. Финальный код на Dart

```dart
String compressIds(List<int> ids) {
  // Граничный случай
  if (ids.isEmpty) return '';

  // Шаг 1: сортируем
  final sorted = List<int>.from(ids)..sort();

  final result = <String>[];
  int start = sorted[0];
  int end = sorted[0];

  for (int i = 1; i < sorted.length; i++) {
    if (sorted[i] == end + 1) {
      // Числа идут подряд — расширяем интервал
      end = sorted[i];
    } else {
      // Разрыв — закрываем текущий интервал
      result.add(_formatRange(start, end));
      // Начинаем новый
      start = sorted[i];
      end = sorted[i];
    }
  }

  // Не забыть добавить последний интервал!
  result.add(_formatRange(start, end));

  return result.join(',');
}

String _formatRange(int start, int end) {
  if (start == end) return '$start';         // одиночка
  if (end == start + 1) return '$start,$end'; // два соседних — НЕ интервал? 
  return '$start-$end';                       // настоящий интервал
}
```

> ⚠️ **Важный момент:** нужно уточнить у интервьюера — два соседних числа (например 3 и 4) писать как "3-4" или "3,4"?  
> В примерах из задачи: `[3,1,2,4] → "1-3,4"` — здесь 3 и 4 НЕ объединяются в интервал.  
> Значит интервал пишем только если длина ≥ 3? Или всё же ≥ 2?  
> Посмотрим на пример: `[9, 8] → "8-9"` — да, два соседних тоже объединяются.  
> Итого: интервал пишем при длине ≥ 2.

Исправленный `_formatRange`:
```dart
String _formatRange(int start, int end) {
  return start == end ? '$start' : '$start-$end';
}
```

## 4. Полный финальный код

```dart
String compressIds(List<int> ids) {
  if (ids.isEmpty) return '';

  final sorted = List<int>.from(ids)..sort();
  final result = <String>[];

  int start = sorted[0];
  int end = sorted[0];

  for (int i = 1; i < sorted.length; i++) {
    if (sorted[i] == end + 1) {
      end = sorted[i];
    } else {
      result.add(start == end ? '$start' : '$start-$end');
      start = sorted[i];
      end = sorted[i];
    }
  }

  result.add(start == end ? '$start' : '$start-$end');

  return result.join(',');
}
```

## 5. Трассировка на примере вручную

**Вход:** `[1, 4, 5, 2, 3, 9, 8, 11, 0]`

После сортировки: `[0, 1, 2, 3, 4, 5, 8, 9, 11]`

```
start=0, end=0

i=1: sorted[1]=1 == end+1=1  →  end=1
i=2: sorted[2]=2 == end+1=2  →  end=2
i=3: sorted[3]=3 == end+1=3  →  end=3
i=4: sorted[4]=4 == end+1=4  →  end=4
i=5: sorted[5]=5 == end+1=5  →  end=5
i=6: sorted[6]=8 ≠ end+1=6  →  сохраняем "0-5", start=8, end=8
i=7: sorted[7]=9 == end+1=9  →  end=9
i=8: sorted[8]=11 ≠ end+1=10 → сохраняем "8-9", start=11, end=11

После цикла: сохраняем "11"

result = ["0-5", "8-9", "11"]
Ответ: "0-5,8-9,11" ✓
```

## 6. Частые ошибки на собесе

**Ошибка 1: Забыть добавить последний интервал после цикла.**
```dart
// Это забывают в 80% случаев!
result.add(start == end ? '$start' : '$start-$end'); // ← вне цикла
```
Последний интервал закрывается только после выхода из цикла — внутри цикла он не добавляется.

**Ошибка 2: Менять исходный список.**
```dart
ids.sort(); // ← плохо, меняем входные данные
final sorted = List<int>.from(ids)..sort(); // ← хорошо
```
На собесе всегда уточняй: "Можно ли сортировать исходный массив?"

**Ошибка 3: Не обработать пустой список.**
```dart
if (ids.isEmpty) return '';
```
Без этого `sorted[0]` упадёт с RangeError.

**Ошибка 4: Неправильно форматировать одиночный элемент.**
```dart
// Одиночка 11 → "11", не "11-11"
start == end ? '$start' : '$start-$end'
```

## 8. Сложность

| Операция   | Сложность  |
|------------|------------|
| Время      | O(n log n) |
| Память     | O(n)       |

Память O(n) — под отсортированную копию и под результат. Если сортировать on-place разрешено — можно считать O(1) дополнительной памяти (не считая результата).

## 9. Follow-up вопросы от Яндекса

**"А если числа уже отсортированы?"**
→ Тогда O(n) по времени — сортировку убираем.

**"А если диапазон очень большой — например [0, 1, 2, ..., 10^9]?"**
→ Алгоритм всё равно O(n) где n — количество элементов в списке, не размер диапазона. Результирующая строка будет просто "0-1000000000".

**"Как изменится решение если нужно десериализовать обратно — из строки в список?"**
→ Обратная операция: сплит по запятой, для каждого элемента если содержит '-' → range(start, end+1), иначе просто число.

```dart
List<int> expandIds(String compressed) {
  if (compressed.isEmpty) return [];
  
  final result = <int>[];
  for (final part in compressed.split(',')) {
    if (part.contains('-')) {
      final bounds = part.split('-');
      final start = int.parse(bounds[0]);
      final end = int.parse(bounds[1]);
      for (int i = start; i <= end; i++) {
        result.add(i);
      }
    } else {
      result.add(int.parse(part));
    }
  }
  return result;
}
```
