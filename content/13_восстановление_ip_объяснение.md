# Задача: Восстановить IP-адрес

> **Уровень сложности:** Medium  
> **Тема:** Бэктрекинг, рекурсия  
> **Аналог на LeetCode:** 93. Restore IP Addresses  
> **Встречается на:** Яндекс, Google, Amazon

---

## 1. Что просят?

```
Дана строка из цифр.
Найти все возможные IPv4-адреса, которые при удалении точек дали бы эту строку.

IPv4 — это 4 числа от 0 до 255, разделённых точками.
Ведущие нули запрещены (кроме самого нуля "0").

Примеры:
  "25525511135"  →  ["255.255.11.135", "255.255.111.35"]
  "0000"         →  ["0.0.0.0"]
  "1111"         →  ["1.1.1.1"]
  "010010"       →  ["0.10.0.10", "0.100.1.0"]
  "1"            →  []   (слишком короткая)
  "111111111111" →  []   (слишком длинная)
```

---

## 2. Как думать на собесе

### Шаг 1: Распознай тип задачи
> "Мне нужно расставить 3 точки в строку так, чтобы получились 4 валидных числа.
> Это перебор с отсечениями — бэктрекинг."

### Шаг 2: Определи ограничения
Каждая из 4 частей:
- Длина от 1 до 3 символов
- Значение от 0 до 255
- Без ведущих нулей (кроме "0")

Из этого сразу видно: строка должна быть длиной от 4 до 12 символов. Иначе ответ — пустой список.

### Шаг 3: Опиши рекурсию
```
backtrack(позиция в строке, текущие части):
  если набрали 4 части и позиция = конец строки → нашли ответ
  если набрали 4 части но строка не кончилась → отсечение
  
  пробуем взять 1, 2 или 3 символа начиная с текущей позиции:
    если подстрока валидна → добавляем в части, рекурсим
    убираем из частей (backtrack)
```

## 3. Код на Dart

```dart
List<String> restoreIpAddresses(String s) {
  // Быстрая проверка длины
  if (s.length < 4 || s.length > 12) return [];

  final result = <String>[];
  _backtrack(s, 0, [], result);
  return result;
}

void _backtrack(String s, int start, List<String> parts, List<String> result) {
  // Базовый случай: набрали 4 части и использовали всю строку
  if (parts.length == 4 && start == s.length) {
    result.add(parts.join('.'));
    return;
  }

  // Отсечение: уже 4 части, но строка не кончилась
  if (parts.length == 4) return;

  // Пробуем взять 1, 2 или 3 символа
  for (int len = 1; len <= 3; len++) {
    if (start + len > s.length) break; // за границу не выходим

    final segment = s.substring(start, start + len);

    if (_isValid(segment)) {
      parts.add(segment);
      _backtrack(s, start + len, parts, result);
      parts.removeLast(); // откатываемся
    }
  }
}

bool _isValid(String segment) {
  // Ведущий ноль запрещён (кроме "0")
  if (segment.length > 1 && segment[0] == '0') return false;
  // Значение должно быть 0-255
  final value = int.parse(segment);
  return value >= 0 && value <= 255;
}
```

## 4. Трассировка на примере

**Вход:** `"1111"`

```
backtrack("1111", 0, []):
  len=1: segment="1" валид → parts=["1"]
    backtrack("1111", 1, ["1"]):
      len=1: segment="1" → parts=["1","1"]
        backtrack("1111", 2, ["1","1"]):
          len=1: segment="1" → parts=["1","1","1"]
            backtrack("1111", 3, ["1","1","1"]):
              len=1: segment="1" → parts=["1","1","1","1"]
                backtrack("1111", 4, ["1","1","1","1"]):
                  parts.length==4, start==4 → ОТВЕТ "1.1.1.1" ✓
```

---

## 5. Трассировка с ведущим нулём

**Вход:** `"010010"`

```
segment="01" → невалид (ведущий ноль) ← отсечение!
segment="0"  → валид
segment="10" → валид
segment="010"→ невалид (ведущий ноль)
```

---

## 6. Частые ошибки

**Ошибка 1: Не проверить ведущие нули.**
```dart
// Плохо — "01" пройдёт как валидный
return int.parse(segment) <= 255;

// Хорошо
if (segment.length > 1 && segment[0] == '0') return false;
```

**Ошибка 2: Не добавить отсечение по длине.**  
Без `if (s.length < 4 || s.length > 12) return []` перебор будет лишним.

**Ошибка 3: Забыть `parts.removeLast()` — откат.**  
Бэктрекинг без отката — не бэктрекинг, просто перебор с багом.

## 8. Сложность

Максимум 3 варианта на каждом из 4 шагов → O(3⁴) = O(81) итераций.  
По сути константное время, так как вход ограничен 12 символами.

| | Сложность |
|---|---|
| Время | O(1) — ограничено константой |
| Память | O(1) — глубина рекурсии максимум 4 |
