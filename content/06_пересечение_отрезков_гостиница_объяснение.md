# Задача: Посетитель гостиницы / Пересекающиеся встречи

> **Уровень сложности:** Medium  
> **Тема:** Интервалы, сортировка, sweep line  
> **Аналог на LeetCode:** 253. Meeting Rooms II  
> **Встречается на:** Яндекс, Google, Facebook

---

## 1. Что просят?

### Вариант А — Посетитель гостиницы
```
Дан список пар (заезд, выезд) для гостей.
Гость выезжает раньше чем следующий заезжает — считается что они не пересекаются.
Найти максимальное количество гостей одновременно в гостинице.

Пример:
  [(1,4), (2,5), (6,8), (3,7)]  →  3
  (гости 1,2,4 одновременно с 3 по 4)
```

### Вариант Б — Пересекающиеся встречи
```
Дан список встреч (from, to).
Вернуть список встреч, которые пересекаются хотя бы с одной другой.

Встречи пересекаются если: !(m1.to <= m2.from || m2.to <= m1.from)
```

---

## 2. Как думать на собесе — Вариант А (максимум гостей)

### Шаг 1: Ключевое наблюдение
> "Когда гость заезжает — счётчик +1. Когда выезжает — счётчик -1.
> Мне нужно найти максимум этого счётчика в любой момент времени."

### Шаг 2: Алгоритм sweep line
```
1. Создать список событий: каждый заезд = +1, каждый выезд = -1
2. Отсортировать по времени (при равном времени: выезд раньше заезда)
3. Пройти по событиям, отслеживать текущий счётчик и максимум
```

### Шаг 3: Важный нюанс — порядок при одинаковом времени
Гость выезжает в момент T, другой заезжает в момент T — они НЕ встречаются.  
Значит при сортировке выезд (-1) должен идти РАНЬШЕ заезда (+1).

## 3. Код на Dart — Вариант А

```dart
int maxGuests(List<(int, int)> intervals) {
  // Разбиваем на события: (время, тип) где тип: +1 = заезд, -1 = выезд
  final events = <(int, int)>[];
  
  for (final (from, to) in intervals) {
    events.add((from, 1));   // заезд
    events.add((to, -1));    // выезд
  }

  // Сортируем: по времени, при равном — выезд (-1) раньше заезда (+1)
  events.sort((a, b) {
    if (a.$1 != b.$1) return a.$1.compareTo(b.$1);
    return a.$2.compareTo(b.$2); // -1 < +1
  });

  int current = 0;
  int maxCount = 0;

  for (final (_, type) in events) {
    current += type;
    if (current > maxCount) maxCount = current;
  }

  return maxCount;
}
```

## 4. Трассировка — Вариант А

**Вход:** `[(1,4), (2,5), (6,8), (3,7)]`

```
События: (1,+1), (4,-1), (2,+1), (5,-1), (6,+1), (8,-1), (3,+1), (7,-1)

После сортировки:
(1,+1), (2,+1), (3,+1), (4,-1), (5,-1), (6,+1), (7,-1), (8,-1)

Проход:
  (1,+1): current=1, max=1
  (2,+1): current=2, max=2
  (3,+1): current=3, max=3  ← пик!
  (4,-1): current=2
  (5,-1): current=1
  (6,+1): current=2
  (7,-1): current=1
  (8,-1): current=0

Ответ: 3 ✓
```

## 5. Код на Dart — Вариант Б (пересекающиеся встречи)

```dart
class Meeting {
  final int from;
  final int to;
  Meeting(this.from, this.to);
  
  bool isCrossing(Meeting other) {
    return !(other.to <= from || other.from >= to);
  }
}

List<Meeting> findCrossing(List<Meeting> meetings) {
  // O(n²) — наивное, но часто принимают если объяснить
  final result = <Meeting>{};
  
  for (int i = 0; i < meetings.length; i++) {
    for (int j = i + 1; j < meetings.length; j++) {
      if (meetings[i].isCrossing(meetings[j])) {
        result.add(meetings[i]);
        result.add(meetings[j]);
      }
    }
  }
  
  return result.toList();
}
```

Оптимальное решение — sweep line за O(n log n):
```dart
List<Meeting> findCrossingOptimal(List<Meeting> meetings) {
  // Сортируем по началу
  final sorted = List<Meeting>.from(meetings)
    ..sort((a, b) => a.from.compareTo(b.from));
  
  final result = <Meeting>{};
  
  // Активные встречи (ещё не закончившиеся)
  // Используем очередь с приоритетом по времени окончания
  final active = <Meeting>[];  // отсортированный по 'to'
  
  for (final meeting in sorted) {
    // Убираем завершившиеся
    active.removeWhere((m) => m.to <= meeting.from);
    
    // Все оставшиеся в active пересекаются с текущей
    if (active.isNotEmpty) {
      result.addAll(active);
      result.add(meeting);
    }
    
    active.add(meeting);
  }
  
  return result.toList();
}
```

## 6. Частые ошибки

**Ошибка 1: Неправильный порядок сортировки при совпадении времён.**  
Если гость выезжает и заезжает в одно время — они НЕ встречаются. Выезд должен обрабатываться первым.

**Ошибка 2: Забыть что выезд это -1, заезд это +1.**  
Перепутать знаки — максимум будет неверным.

**Ошибка 3: Считать максимум только в конце.**  
Максимум нужно обновлять после каждого события.
