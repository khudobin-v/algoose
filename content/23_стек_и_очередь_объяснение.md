# Задача: Реализация стека и очереди на массивах

> **Уровень сложности:** Easy-Medium  
> **Тема:** Структуры данных, массивы  
> **Встречается на:** Яндекс, все крупные компании (базовый вопрос)

---

## 1. Что просят?

Реализовать стек (LIFO) и очередь (FIFO) на массивах фиксированного размера.

---

## 2. Стек (Stack, LIFO)

**Ключевая идея:** указатель `top` на верхний элемент. `push` = добавить и top++. `pop` = взять и top--.

```
Операция | Действие
push(x)  | data[++top] = x
pop()    | return data[top--]
peek()   | return data[top]
isEmpty  | top == -1
isFull   | top == capacity-1
```

**Псевдокод push/pop:**
```
push(x): если full → ошибка; data[++top] = x
pop():   если empty → ошибка; val = data[top]; top--; return val
```

---

## 3. Очередь (Queue, FIFO) — кольцевой буфер

**Ключевая идея:** два указателя `head` и `tail`. Используем `% capacity` для зацикливания.

```
Операция    | Действие
enqueue(x)  | data[tail] = x; tail = (tail+1) % capacity; size++
dequeue()   | val = data[head]; head = (head+1) % capacity; size--; return val
isEmpty     | size == 0
isFull      | size == capacity
```

**Почему кольцевой буфер?**
Без кольца: после нескольких enqueue/dequeue указатели "уйдут" к концу массива и место в начале не используется.
С кольцом `(index + 1) % capacity`: указатели оборачиваются и используют весь массив.

---

## 4. Сложность операций

| Операция | Стек | Очередь |
|---|---|---|
| push/enqueue | O(1) | O(1) |
| pop/dequeue | O(1) | O(1) |
| peek | O(1) | O(1) |

---

## 5. Типичные ошибки

**Ошибка 1: Различение пустой и полной очереди.**
Если хранить только head и tail без size — невозможно отличить пустую от полной (оба head==tail).
Решение: хранить отдельный счётчик `size`.

**Ошибка 2: Не очищать ячейку после pop/dequeue.**
Для GC языков (Dart, Java) важно обнулять ссылку: `data[head] = null`.

**Ошибка 3: Не проверять переполнение и опустошение.**
Всегда бросать исключение при push в полный стек или pop из пустого.
