# Задача: Сложение двух hex-чисел строками

> **Уровень сложности:** Easy-Medium  
> **Тема:** Строки, сложение столбиком  
> **Встречается на:** Яндекс

---

## 1. Что просят?

```
Сложить два шестнадцатеричных числа заданных строками.

Примеры:
  "1a" + "2b" = "45"    (26+43=69=0x45)
  "ff" + "1"  = "100"   (255+1=256)
  "0"  + "0"  = "0"
```

---

## 2. Ключевая идея

Сложение столбиком в шестнадцатеричной системе счисления.
Идём с конца обеих строк. На каждом шаге: сумма двух цифр + перенос, новый перенос = сумма / 16.

---

## 3. Пошаговый алгоритм

```
i = len(a)-1, j = len(b)-1, carry = 0

Пока i>=0 ИЛИ j>=0 ИЛИ carry>0:
  dA = i>=0 ? hexToInt(a[i]) : 0
  dB = j>=0 ? hexToInt(b[j]) : 0
  sum = dA + dB + carry
  carry = sum / 16
  result.prepend(intToHex(sum % 16))
  i--, j--

Вернуть result
```

---

## 4. Трассировка

**a="ff", b="1"**

```
i=1('f'), j=0('1'), carry=0:
  dA=15, dB=1, sum=16, carry=1, digit=0 → result="0"
i=0('f'), j=-1, carry=1:
  dA=15, dB=0, sum=16, carry=1, digit=0 → result="00"
i=-1, j=-1, carry=1:
  sum=0+0+1=1, carry=0, digit=1 → result="100"

Ответ: "100" ✓
```

---

## 5. Сложность

| | Сложность |
|---|---|
| Время | O(max(len(a), len(b))) |
| Память | O(max(len(a), len(b))) |

---

## 6. Типичные ошибки

**Ошибка 1: Забыть обработать carry после окончания обоих строк.**
Цикл должен продолжаться пока `carry > 0`.

**Ошибка 2: Неправильно строить результат.**
Цифры строятся от младшего к старшему — нужно перевернуть строку в конце.

**Ошибка 3: Не обработать символы 'A'-'F' (заглавные).**
Привести к нижнему регистру через `toLowerCase()`.
