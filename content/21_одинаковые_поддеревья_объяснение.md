# Задача: Найти эквивалентные поддеревья

> **Уровень сложности:** Medium  
> **Тема:** Деревья, сериализация, HashMap  
> **Аналог на LeetCode:** 652. Find Duplicate Subtrees  
> **Встречается на:** Яндекс, Google

---

## 1. Что просят?

```
В бинарном дереве найти все поддеревья, которые встречаются более одного раза.
Два поддерева считаются одинаковыми если у них одинаковая структура
и одинаковые значения во всех узлах.

Пример:
        1
       / \
      2   3
     / \ /
    4  5 2
        / \
       4   5
Поддерево (2,4,5) встречается дважды → ответ [узел 2 (любой из двух)]
```

---

## 2. Ключевая идея

Сериализовать каждое поддерево в строку-дескриптор.
Одинаковые поддеревья → одинаковые дескрипторы.
Считать частоты дескрипторов через HashMap.

Дескриптор: `"val,left_descriptor,right_descriptor"`, пустой узел = `"#"`.

---

## 3. Пошаговый алгоритм

```
Рекурсивная функция serialize(node):
  если node == null: вернуть "#"
  
  left  = serialize(node.left)
  right = serialize(node.right)
  descriptor = "node.val,left,right"
  
  count[descriptor]++
  если count[descriptor] == 2: добавить node в result
  
  вернуть descriptor
```

---

## 4. Пример дескрипторов

Для узла 4 (лист): `"4,#,#"`
Для узла 5 (лист): `"5,#,#"`
Для узла 2 (с детьми 4,5): `"2,4,#,#,5,#,#"`

Если два узла имеют одинаковый дескриптор — поддеревья эквивалентны.

---

## 5. Сложность

| | Сложность |
|---|---|
| Время | O(n²) — из-за конкатенации строк; O(n) с хешированием |
| Память | O(n) — под дескрипторы |

---

## 6. Типичные ошибки

**Ошибка 1: Добавлять узел в результат при каждом дубликате.**
Добавлять только при `count == 2`, иначе один и тот же узел попадёт несколько раз.

**Ошибка 2: Использовать inorder-обход для сериализации.**
Inorder не уникально определяет дерево. Нужен постфиксный (post-order) обход с явными нулями.

**Ошибка 3: Сравнивать узлы по ссылке, а не по содержимому.**
Два разных объекта с одинаковой структурой должны давать одинаковую строку.
